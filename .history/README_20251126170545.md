# Asset Management System

Full-stack application for managing company assets and IT support tickets.

## Technologies
- **Frontend**: Vue.js 3 + Vite
- **Backend**: ASP.NET Core 6 Minimal API
- **Database**: SQL Server / SQL Server Express

## Prerequisites

1. **Node.js** (v16 or higher) - [Download](https://nodejs.org/)
2. **SQL Server** or **SQL Server Express** - [Download](https://www.microsoft.com/en-us/sql-server/sql-server-downloads)
3. **.NET 6 SDK** - [Download](https://dotnet.microsoft.com/download/dotnet/6.0)

## Setup Instructions

### 1. Clone the Repository
```bash
git clone https://github.com/ENG02222222/AssetManagement_EngXianYu.git
cd AssetSolution
```

### 2. Database Setup

#### Find Your SQL Server Instance Name:
Open SQL Server Management Studio (SSMS) and note the server name when connecting.
Common formats:
- `localhost\SQLEXPRESS`
- `.\SQLEXPRESS`
- `YOUR_COMPUTER_NAME\SQLEXPRESS`
(Mine is Server=LAPTOP-KKU05C2B\\SQLEXPRESS)

#### Create the Database:
1. Open SSMS
2. Connect to your SQL Server
3. Open the file `AssetAPI/setup-database.sql`
4. Execute the script (or use the migration command below if using EF)

**OR** use Entity Framework:
```bash
cd AssetAPI
dotnet ef database update
```

### 3. Backend Setup

#### Configure Connection String:
1. Copy `appsettings.Example.json` to `appsettings.Development.json`
2. Update the connection string with YOUR server name:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=YOUR_SERVER_NAME\\SQLEXPRESS;Database=AssetDB;Integrated Security=True;TrustServerCertificate=True;"
  }
}
```

#### Run Backend:
```bash
cd AssetAPI
dotnet restore
dotnet run
```

Backend will run on:
- `http://localhost:5011`
- `https://localhost:7130`

### 4. Frontend Setup

#### Install Dependencies:
```bash
cd ../AssetVue  # or your frontend folder name
npm install
```

#### Configure API URL:
1. Copy `.env.example` to `.env`
2. Verify the API URL:
```
VITE_API_BASE_URL=http://localhost:5011
```

#### Run Frontend:
```bash
npm run dev
```

Frontend will run on: `http://localhost:5173`

## Troubleshooting

### Backend won't start - SQL Connection Error
- Make sure SQL Server service is running (services.msc)
- Verify your server name in `appsettings.Development.json`
- Test connection in SSMS first

### Frontend can't connect to backend
- Check backend is running on port 5011
- Verify `.env` file has correct API URL
- Check browser console for CORS errors

### Port already in use
```bash
# Kill existing dotnet processes
taskkill /IM dotnet.exe /F
```
